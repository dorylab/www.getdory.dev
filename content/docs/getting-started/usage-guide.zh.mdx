# Dory 使用文档

Dory 是面向 SQL 和 AI 的数据工作台。典型使用路径是：

1. 打开 Dory。
2. 创建或选择数据连接。
3. 查看表和字段。
4. 运行 SQL。
5. 检查结果表。
6. 当结果结构适合可视化时切换到 Charts。
7. 需要继续分析或复用时使用 AI 和 Saved Queries。

这篇文档按当前产品里的真实流程组织。

## 1. 打开 Dory

你可以通过桌面 App、托管产品或自托管部署使用 Dory。

如果要访问 SQLite、DuckDB、CSV、Excel、Parquet 这类本地文件，建议使用桌面 App，或者确保自托管服务能访问对应文件路径。填写的路径必须能被 Dory 运行时读取。Dory 会为当前用户自动准备默认工作上下文，打开后可以直接连接数据。

## 2. 连接数据

打开 **Connections**。这里可以创建数据库连接，也可以打开本地文件数据集。

### 数据库连接

点击 **Add Connection**，选择数据库类型，填写必填配置，然后创建连接。

| 类型 | 适用场景 |
| --- | --- |
| ClickHouse | 日志、事件、可观测性、分析和监控工作流。 |
| PostgreSQL / Neon | 业务数据、产品分析和通用 SQL 工作。 |
| MySQL / MariaDB | 业务系统、Web 应用和报表。 |
| SQLite | 本地 `.sqlite` / `.db` 文件、Demo、原型和可复现实验。 |
| DuckDB / MotherDuck | 本地分析文件、嵌入式分析和 MotherDuck 项目。 |

在 Dory 桌面端连接本地 SQLite 文件时，选择 **SQLite**，并填写绝对路径，例如：

```text
/Users/Shared/Data/demo.sqlite
```

![Dory 桌面端 SQLite 连接](/images/core-features/dory-desktop-connections-sqlite.png)

### Open Files

当数据源是 Excel、CSV、Parquet 这类文件，并且你希望 Dory 把它们作为可查询表暴露出来时，使用 **Open Files**。

流程是：

1. 点击 **Open Files**。
2. 填写数据集名称。
3. 填写服务端可访问的 **File Path**。
4. 点击 **Preview**。
5. 查看识别到的 sheets 或文件关系。
6. 保留要导入的 sheet/table，或展开 **Advanced options** 自定义表名。
7. 创建数据集。

创建完成后，这个数据集会出现在 Connections 中，并可在 Explorer 和 SQL Console 中使用。

## 3. 查看数据结构

写 SQL 前，先确认数据结构。

需要完整对象视图时，使用 **Explorer**。它适合查看数据库、Schema、表、视图、字段、DDL、数据预览和表详情。

![Dory Explorer 表列表](/images/getting-started/dory-usage-explorer.png)

已经在写 SQL 时，可以直接使用 SQL Console 左侧 Schema 面板。它会显示当前数据库、表列表，并提供插入起始 `SELECT` 的快捷操作。

![Dory Explorer 表数据预览](/images/getting-started/dory-usage-explorer-table-preview.png)

分析前建议确认：

- 当前数据库或 Schema 是否正确。
- 目标表是否存在。
- 字段名和字段类型是否符合预期。
- 预览数据是否是你要分析的数据。

## 4. 运行 SQL

选中正确连接后，打开 **SQL Console**。

第一次探索可以先运行小范围预览：

```sql
select *
from orders
limit 100;
```

进入分析时，先把结果整理成适合阅读或图表的结构。例如：

```sql
select strftime('%Y-%m', created_at) as month,
       status,
       round(sum(amount), 2) as revenue,
       count(*) as orders
from orders
group by month, status
order by month, status;
```

运行后重点检查：

- 查询是否成功完成。
- 执行耗时。
- 返回行数。
- 列名和值是否正确。
- 输出是否回答了当前分析问题。

![Dory 桌面端 SQL Console SQLite 结果](/images/core-features/dory-desktop-sql-console-sqlite-results.png)

## 5. 查看结果并生成图表

查询结果会出现在 SQL 编辑器下方。

先用 **Table** 检查原始值。只有当结果已经适合可视化时，再切换到 **Charts**。

常见图表输入结构：

| 目标 | SQL 输出结构 |
| --- | --- |
| 趋势 | 时间列 + 数值指标。 |
| 对比 | 分类列 + 数值指标。 |
| 分组趋势 | 时间列 + 分组列 + 数值指标。 |
| 明细检查 | ID、时间、状态和必要的诊断字段。 |

如果图表不理想，优先修改 SQL。通常需要提前聚合、明确列名、减少无关字段，或按时间/分类排序。

![Dory 桌面端 SQL Console SQLite 图表](/images/core-features/dory-desktop-sql-console-sqlite-charts.png)

## 6. 使用 AI 辅助

Dory 有两个 AI 入口。左侧导航的 **Chatbot** 适合围绕当前连接做开放式提问；SQL Console 里的 **Ask AI** 适合基于当前 SQL、查询结果或后续图表分析继续处理。

适合使用 AI 的场景：

- 用自然语言生成 SQL。
- 解释陌生 SQL。
- 修复 SQL 报错。
- 把结果改写成更适合图表的结构。
- 基于当前结果继续分析。

![Dory AI Chat 问题与回答](/images/getting-started/dory-usage-ai-chat-question-answer.png)

AI 生成的 SQL 需要先审核再执行，尤其是生产数据库。如果目标是图表，可以明确要求 AI 输出包含时间/分类列和指标列的 chart-ready result。

## 7. 保存可复用查询

当一条查询已经验证有效，并且后续可能复用时，点击 **Save Query**。

建议保存：

- 周期性指标查询。
- 排查和运营检查 SQL。
- 团队共享分析。
- 图表、看板或固定报告依赖的基础查询。

![Dory Save Query 弹窗](/images/getting-started/dory-usage-save-query.png)

命名应描述业务问题，不要只写表名。例如用 `Monthly revenue by status`，不要只写 `orders query`。

## 8. 使用 ClickHouse 专属流程

如果连接的是 ClickHouse，并且当前账号具备权限，Dory 会提供更深入的能力：

- **Monitoring**：最近查询、慢查询、错误查询、延迟、QPM 和 SQL 详情。
- **Privileges**：用户、角色、全局权限、数据库 / 表 / 视图权限和集群级操作。

生产环境建议使用最小权限账号。Monitoring 和 Privileges 依赖 ClickHouse 系统表与账号权限。

## 9. 接入外部 AI Agent

Dory 支持 MCP / Agent Access，可以让 Codex、Claude 等工具受控访问 Dory 上下文。

常见 MCP 用法：

- 列出可用 Dory 连接。
- 读取 Schema 和表元数据。
- 预览表数据。
- 执行受限制的只读 SQL。
- 在外部 Agent 工作流中使用数据库上下文。

需要配置时，从 [MCP / Agent Access](/zh/docs/reference/mcp) 开始。
