# 3 分以内に最初のクエリ

このガイドは、データベース接続がすでに機能していることを前提としています。

## 始める前に

- 開発データベース、デモデータベース、または読み取り専用データベースを選択します。
- 接続を含むワークスペースを開きます。
- エクスプローラーでテーブルと列を表示できることを確認します。

## 1. エクスプローラーを開いてスキーマを検査する

エクスプローラーで起動します。テーブルを選択し、その列、サンプル行、名前またはキーによって暗示される関係を確認します。

## 2.SQL コンソールを開きます

同じ接続から SQL コンソールを開きます。SQL の書き込み中にテーブルと列の名前を確認できるように、エクスプローラーを使用可能な状態にしておきます。

## 3. 最初の安全なクエリを実行する

限定されたクエリから始めます。

```sql
select *
from your_table
limit 20;
```

フィルター、結合、または集計を作成する前に、結果の列を確認してください。

## 4. AI を使用して次のクエリを生成する

より具体的なクエリについては、Dory AI に尋ねてください。テーブル名、必要なメトリック、フィルター、および予期される出力列を含めます。

例：

過去 7 日間のステータスごとに行をカウントし、ステータスとカウントを返すように AI に依頼します。

## 5. 結果を確認してグラフを作成する

結果が時間、カテゴリ、またはステータスごとに集計されている場合は、結果領域からグラフを開き、データの形状に一致するグラフの種類を選択します。

## 6. 最初の分析を保存する

クエリが役立つ場合は、明確なビジネス名と前提条件の短い説明を付けて保存します。

## 一般的なトラブルシューティング

### クエリは行を返しません

フィルターと時間範囲を確認してください。より小規模な条件セットから始めます。

### クエリは失敗します

AI にエラーの説明を依頼し、提案された修正を Explorer スキーマと比較します。

### 結果が大きすぎます

` limit `を追加するか、時間範囲を狭めるか、行を返す前に集計します。

### オートコンプリートがありません

選択した接続とスキーマが正しいことを確認してください。

### 変更後にクエリが失敗する

変更された SQL を最後に動作していたバージョンと比較し、特定のエラー メッセージのみを AI に説明してもらいます。

### AI の結果は不正確です

AI に正確なテーブル名、列、必要なフィルター、予期される出力形状を与えます。
