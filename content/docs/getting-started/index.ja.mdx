＃ 概要

最初の Dory セッションには短いパスがあり、アプリを起動し、アクセス可能なデータ ソースに接続し、限定されたクエリを実行し、結果を確認します。

## セットアップパス

1. [クイック スタート](/ja/docs/getting-started/quick-start): ホストされたデモ、デスクトップ アプリ、または Docker を選択します。
2. [インストール](/ja/docs/getting-started/installation): デスクトップ アプリをインストールするか、セルフホスト ランタイムを起動します。
3. [データベースに接続](/ja/docs/getting-started/connect-your-database): 接続を作成し、スキーマの可視性を確認します。
4. [3 分以内の最初のクエリ](/ja/docs/getting-started/first-query-in-3- minutes): 限定されたクエリを実行し、結果を確認し、次のステップで AI を使用します。

## セットアップ後の作業

最初のクエリが機能したら、目的に応じてドキュメントの残りの部分を使用します。

| 必要 | 読む |
| --- | --- |
| 製品の表面を理解する | [コア機能](/ja/docs/core-features) |
| タスク指向のプロセスに従う | [ワークフロー](/ja/docs/workflows) |
| データベース固有の動作を確認する | [データベースサポート](/ja/docs/database-support) |
| デプロイメントまたは認証を構成する | [デプロイ](/ja/docs/deploy) |
| 設定、ショートカット、スニペット、MCP、またはFAQを検索します。 | [参考資料](/ja/docs/reference) |

## 最小限の生産に関するアドバイス

- 運用データベースには読み取り専用の資格情報を使用します。
- Dory を実行するランタイム、特に Docker からのネットワーク アクセスを確認します。
- AI が生成した SQL を実行前に確認します。
