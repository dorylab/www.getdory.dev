# AI Chat

Dory AI Chat 是面向数据库工作的 AI 助手。当前产品里有两个入口：

- 左侧导航里的 **Chatbot**：适合围绕当前连接进行完整对话。
- SQL Console 里的 **Ask AI**：适合基于当前 SQL、结果和分析上下文继续处理。

两个入口都会围绕当前选中的连接工作，所以使用前先确认连接是否正确。

## Chatbot 页面

当你还在探索问题、想生成第一版 SQL，或想理解陌生表结构时，打开左侧 **Chatbot**。

![Dory AI Chat 首页](/images/core-features/dory-ai-chat-home.png)

提问后，Dory 会把对话保存在当前 session 中，并在同一页面展示问题和回答。

![Dory AI Chat 问题与回答](/images/core-features/dory-ai-chat-question-answer.png)

更有效的提问通常会明确表、指标、时间范围和输出结构，例如：

- Show top 10 users by order amount.
- Generate daily order trends for this month.
- Summarize all tables with row counts.
- Find error logs in the last 24 hours.

## SQL Console AI 面板

当你已经在 SQL Console 中编写或运行 SQL 时，使用 **Ask AI**。右侧 AI 面板可以结合当前编辑器内容、Schema、最近执行结果和结果结构。

![Dory SQL Console AI 面板](/images/core-features/dory-ai-chat-sql-console-panel.png)

它适合这些场景：

- 解释当前编辑器里的 SQL。
- 修复查询报错。
- 把结果改写成适合 Charts 的结构。
- 基于当前结果继续分析。
- 把临时查询整理成适合保存复用的版本。

## AI Chat 理解哪些上下文

AI Chat 可以使用 Dory 中已经存在的上下文：

- 当前连接和数据库。
- 可见的 Schema、表和字段。
- 当前 SQL 编辑器内容。
- 最近的查询错误或结果结构。
- 当前对话历史。

这些上下文会让 AI 生成的 SQL 更贴近真实 Schema，而不是通用模板。

## 推荐流程

1. 选中正确连接。
2. 先在 **Explorer** 中确认目标表和字段。
3. 如果还没有 SQL，从 **Chatbot** 开始；如果已经在写 SQL，从 SQL Console 的 **Ask AI** 开始。
4. 执行前先审核生成的 SQL。
5. 先用安全的 `LIMIT` 或时间范围运行。
6. 再通过追问让 AI 解释、修复、优化或调整结果结构。

## 最佳实践

- 知道表名时直接写出准确表名。
- 说明指标、维度、时间范围和希望返回的字段。
- 如果要生成图表，明确要求输出 chart-ready result。
- 生产数据库中，把生成 SQL 当作草稿，审核后再执行。
- 除非明确需要写操作，优先让 AI 生成只读查询。
