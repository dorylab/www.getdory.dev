# AI チャット

Dory AI Chat はデータベース作業用に構築されています。次の 2 つの場所で利用できます。

- 左側のサイドバーにある **チャットボット** では、選択した接続に関する自由回答の質問が可能です。
- 現在の SQL タブ、結果、分析コンテキストに関するヘルプについては、SQL コンソールで AI に問い合わせてください**。

どちらのエントリ ポイントもアクティブな接続をコンテキストとして使用するため、まず AI に推論させたい接続を選択します。

## チャットボット ページ

選択したデータベースに関する完全な会話が必要な場合は、**チャットボット**を使用します。これは、最初の SQL ドラフトを生成する場合、次に何を検査するかを尋ねる場合、または馴染みのないテーブルを探索する場合に最適です。

![ドリー AI チャット ホーム](/images/core-features/dory-ai-chat-home.png)

質問すると、ドリーはセッションを継続して、プロンプトの横に回答を表示します。

![Dory AI チャットの質問と回答](/images/core-features/dory-ai-chat-question-answer.png)

適切なプロンプトは、テーブル、メトリック、時間範囲、出力形状に特化しています。

- 注文金額別に上位 10 人のユーザーを表示します。
- 今月の毎日の注文傾向を生成します。
- すべてのテーブルを行数とともに要約します。
- 過去 24 時間のエラー ログを検索します。

## SQL コンソール AI パネル

すでに SQL を作成または実行している場合は、SQL コンソールで **Ask AI** を使用します。このパネルは、現在のエディターのコンテンツ、表示されているスキーマ、最近の実行結果、および結果の形状を操作できます。

![Dory SQL コンソール AI パネル](/images/core-features/dory-ai-chat-sql-console-panel.png)

これは次の場合の最速のパスです。

- 現在エディターにある SQL について説明します。
- クエリエラーを修正しました。
- 結果をチャートに対応できるように書き換えます。
- 現在の結果テーブルから分析を継続します。
- 大まかなクエリを再利用可能な保存済みクエリの候補に変換します。

## AI チャットが理解できること

AI Chat は、Dory ですでに利用可能なコンテキストを使用できます。

- 選択された接続とデータベース。
- スキーマ、テーブル、列が表示されます。
- 現在の SQL エディターのコンテンツ。
- 最近のクエリ エラーまたは結果の形状。
- 現在の会話のチャット履歴。

コンテキストは、AI が一般的な例ではなく実際のスキーマに一致する SQL を生成するのに役立ちます。

## 推奨フロー

1. 正しい接続を選択します。
2. **エクスプローラー**を使用して、対象のテーブルと列を確認します。
3. チャットボットに最初のドラフトを依頼するか、クエリがすでに進行中の場合は、SQL コンソールから **Ask AI** を開きます。
4. 生成された SQL を実行する前に確認します。
5. 最初に安全な`LIMIT `または時間範囲を指定して実行します。
6. フォローアップ プロンプトを使用して、結果を説明、修正、最適化、または再形成します。

## ベストプラクティス

- 正確なテーブル名がわかっている場合は、それを含めます。
- メトリック、ディメンション、時間範囲、および必要な列を記述します。
- チャートを使用する予定がある場合は、チャート対応の出力を要求します。
- 特に運用データベースでは、生成された SQL をドラフトとして扱います。
- 意図的に書き込み操作が必要な場合を除き、読み取り専用クエリを優先します。
