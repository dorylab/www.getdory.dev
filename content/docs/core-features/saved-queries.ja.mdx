# 保存されたクエリ

保存されたクエリにより、有用な SQL が再利用可能な分析に変わります。これらは、再度実行したり、チームメイトと共有したりする予定のクエリに最適です。

## 節約できるもの

- 再利用可能なレポート。
・動作確認。
- クエリのデバッグ。
- チャート対応 SQL。
- クエリは手動レビュー後に AI で生成または改善されます。

## いつ保存するか

- 同じ質問が繰り返し出てきます。
- クエリには、明らかではないビジネス ロジックが含まれています。
- チームメイトがそれを再利用する必要があるかもしれません。
- 結果により、繰り返し発生するグラフやワークフローが強化されます。

## 命名に関する推奨事項

単なるテーブル名ではなく、ビジネス上の質問を説明する名前を使用してください。

良い例:

- 製品分野ごとの週間アクティブ ユーザー。
- 過去 24 時間以内に支払いが失敗した場合。
- ユーザーによる ClickHouse クエリが遅い。

## 利用の流れ

1. SQL を書き込み、検証します。
2. 書式設定をクリーンアップし、一時フィルターを削除します。
3. 明確な名前と説明を追加します。
4. クエリを保存します。
5. 定期的に見直して、仮定を最新の状態に保ちます。

## チーム練習

時間範囲、フィルター、結合、メトリック定義を文書化して、他のユーザーがクエリを安全に再利用できるようにします。
