# ドリーと一緒に未知のテーブルを探索する

新しいデータベースまたはスキーマを開くときは、AI が生成した複雑なクエリから始めないでください。まず、テーブル構造、列タイプ、サンプル行を検査します。次に、AI に関係を説明するか、実際のコンテキストを含む SQL を生成するように依頼します。

## このワークフローを使用する場合

- 新しい ClickHouse、PostgreSQL、MySQL、SQLite、または DuckDB データベースに接続しました。
- どのテーブルにメトリクスが含まれているかがわかりません。
- テーブル名と列名が見慣れない。
- 別のチームからデータセットを継承しました。
- データがグラフの準備ができているかどうかを決定する必要があります。

## 1. エクスプローラーで起動します

[エクスプローラー](/ja/docs/core-features/explorer) を開き、データベース、スキーマ、テーブル、ビューが正しく読み込まれることを確認します。

テーブル名、時間列、ステータス フィールド、ID、数値メトリック、コメント、パーティション キー、並べ替えキーを探します。

## 2. テーブルを分類する

| テーブルタイプ | 一般的な兆候 | 分析方法 |
| --- | --- | --- |
| 詳細表 | ID、タイムスタンプ、ステータス、金額 | 最初にプレビューしてから、時間とディメンションごとに集計します。 |
| 寸法表 | 行数、名前、カテゴリ、属性が少ない | 結合やラベルの説明に使用します。 |
| 集計テーブル | 日付またはメトリック列がすでに存在します | 二重カウントや繰り返しの集計を避けてください。 |
| ログテーブル | 大ボリューム、多くのフィールド、明確な時間列 | 常に時間でフィルターし、行を制限します。 |

## 3. 安全なプレビュー クエリを実行する

```sql
SELECT *
FROM your_table
LIMIT 50;
```

大きなテーブルの場合は、最初に時間フィルターを追加します。

## 4. AI に構造を説明してもらう

```text
Based on the current table schema, explain what business data this table likely contains. Identify time dimensions, measures, and categorical dimensions.
```

## 5. 探索クエリの生成

AI に低リスクの探索クエリを依頼します。

```text
Generate 3 exploration queries for the current table: daily row volume, status distribution, and recent abnormal records. Include a safe time range or LIMIT.
```

生成された SQL を実行する前に確認してください。

## FAQ

### AI が関係を推測できない場合はどうなるでしょうか?

エクスプローラーでテーブル名と列名を確認し、関連する可能性のあるテーブルを明示的に指定します。

### プレビュー クエリが遅い場合はどうすればよいですか?

クエリを実行する前に、時刻、パーティション、またはソート キーを見つけて範囲を絞ります。

### 運用クエリの作成はいつから始められますか?

テーブル粒度、重要な列、時間範囲のセマンティクス、および安全なフィルターを理解してから始めてください。最初に小さなプレビュー クエリを実行して仮定を検証します。
