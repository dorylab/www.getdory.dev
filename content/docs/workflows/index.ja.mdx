＃ 概要

ワークフローには、Dory の最も一般的な分析タスクが文書化されています。各ガイドは、接続、エクスプローラー、SQL コンソール、AI チャット、グラフと結果、保存されたクエリを反復可能なプロセスに接続します。

## 推奨されるワークフロー

| ワークフロー | 使用事例 | 主な特長 |
| --- | --- | --- |
| [不明なテーブルの探索](/ja/docs/workflows/explore-unknown-tables) | 馴染みのないスキーマを理解する | エクスプローラー、テーブルプレビュー、AI チャット |
| [AI を使用して SQL を生成](/ja/docs/workflows/generate-sql-with-ai) | ビジネスの質問を SQL に変換します | AI チャット、SQL コンソール、スキーマ コンテキスト |
| [遅いクエリのデバッグ](/ja/docs/workflows/debug-slow-queries) | 高価な SQL を調査する | SQL コンソール、AI チャット、ClickHouse モニタリング |
| [SQL からチャートを作成](/ja/docs/workflows/build-charts-from-sql) | 視覚化のためにクエリ出力を整形する | SQL コンソール、チャートと結果、AI |
| [分析の保存と再利用](/ja/docs/workflows/save-reuse-analysis) | 有用なクエリとチームの知識を保存する | 保存されたクエリ、フォルダー、説明 |

## 推奨される学習パス

1. [Explore Unknown Tables](/ja/docs/workflows/explore-unknown-tables) を使用してスキーマを探索します。
2. [Generate SQL with AI](/ja/docs/workflows/generate-sql-with-ai) を使用して最初のクエリを生成します。
3. [SQL Console](/ja/docs/core-features/sql-console) でクエリを実行し、調整します。
4. [Debug Slow Queries](/ja/docs/workflows/debug-slow-queries) を使用してパフォーマンスをデバッグします。
5. [Build Charts from SQL](/ja/docs/workflows/build-charts-from-sql) で安定した出力を視覚化します。
6. [分析の保存と再利用](/ja/docs/workflows/save-reuse-analysis) を使用して、再利用可能な作業を保存します。
