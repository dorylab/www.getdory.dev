＃ 概要

共有 Dory デプロイメントは通常、ランタイム、永続性、パブリックURL、認証、AI プロバイダー、データベース ネットワーク、および秘密管理の決定から始まります。

## クイックリンク

| ページ | 使用事例 | 集中 |
| --- | --- | --- |
| [セルフホスティング](/ja/docs/deploy/self-hosting) | チーム展開を計画している場合 | アーキテクチャの選択、永続性の決定、ロールアウトのチェック、および運用の境界。 |
| [Docker](/ja/docs/deploy/docker) | 具体的なコンテナコマンドが必要です | `docker run `、`.env `、ポート、ボリューム、Postgres モード、およびアップグレード。 |
| [環境変数](/ja/docs/deploy/environment-variables) | 構成リファレンスが必要です | 必要なシークレット、アプリケーション データベース、AI、認証、電子メール、OAuth、および関連変数。 |
| [認証](/ja/docs/deploy/auth-sso) | ユーザーがサインインする方法を決定するのはあなたです | 電子メールによるログイン、検証、OAuth、初期管理、プロビジョニング、オフボード。 |

## 推奨される順序

1. [セルフホスティング](/ja/docs/deploy/self-hosting) を読んで、ストレージ、ネットワーク、HTTPS、認証、およびロールアウト境界を選択します。
2. 実際のコンテナ起動コマンドは[Docker](/ja/docs/deploy/docker)を使用します。
3. [環境変数](/ja/docs/deploy/environment-variables) から必要な値を入力します。
4. [Auth](/ja/docs/deploy/auth-sso) でログイン ルールを設定します。
5. 運用接続を追加する前に、テスト データベースを使用して検証します。

## 起動チェックリスト

- パブリック アクセスはHTTPSを使用します。
- ` BETTER_AUTH_URL `は、ユーザーが開いているURLと一致します。
- アプリケーション データはボリュームまたは Postgres を通じて保持されます。
- 初期管理者、ログイン方法、オフボードプロセスが決定されます。
- 運用データベース接続では、最初に読み取り専用の資格情報が使用されます。
