# 概览

团队部署通常需要先确定运行环境、持久化方式、公开访问地址、认证策略、AI Provider、数据库网络和 Secret 管理方式。

## 快速入口

| 页面 | 适用场景 | 重点内容 |
| --- | --- | --- |
| [自托管部署](/zh/docs/deploy/self-hosting) | 准备团队部署 | 架构选择、持久化决策、上线检查和运维边界。 |
| [Docker 部署](/zh/docs/deploy/docker) | 需要具体容器命令 | `docker run`、`.env`、端口、volume、Postgres 模式和升级。 |
| [环境变量配置](/zh/docs/deploy/environment-variables) | 查询完整配置项 | 必填 Secret、应用数据库、AI、认证、邮件、OAuth 等变量。 |
| [认证配置](/zh/docs/deploy/auth-sso) | 规划用户如何登录 | 邮箱登录、邮箱验证、OAuth、初始化管理员、成员开通和离职回收。 |

## 推荐阅读顺序

1. 先读 [自托管部署](/zh/docs/deploy/self-hosting)，确定存储、网络、HTTPS、认证和上线边界。
2. 用 [Docker 部署](/zh/docs/deploy/docker) 执行具体启动命令。
3. 按 [环境变量配置](/zh/docs/deploy/environment-variables) 补齐必填配置。
4. 在 [认证配置](/zh/docs/deploy/auth-sso) 中确定登录策略。
5. 先接入测试数据库验证，再接入生产连接。

## 上线检查

- 公开访问地址已经启用 HTTPS。
- `BETTER_AUTH_URL` 与用户实际访问地址一致。
- 应用数据已经持久化到 volume 或 Postgres。
- 初始化管理员、登录方式和成员回收流程已经确定。
- 生产数据库优先使用只读账号。
