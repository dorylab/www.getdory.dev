＃ 設定

Dory 設定は、各ユーザーのローカル ワークフロー設定とチームの共有操作選択の 2 つの層をカバーします。 Dory SQL ワークスペースを標準化する場合、またはオンボーディングのデフォルトを文書化する場合は、このリファレンスを使用してください。

## SQL エディターの設定

| 設定 | オプション | デフォルト | 影響を受けるもの |
| --- | --- | --- | --- |
| エディターのテーマ | `Auto `、` GitHub Light `、` GitHub Dark `、` VS Light `、` VS Dark ` | ` Auto ` | モナコエディターの色とコントラスト。 |
| フォントファミリー | ` Monaco `、` Menlo `、` Consolas `、` JetBrains Mono `、` Fira Code `、` Source Code Pro `、` Custom ` | ` Monaco ` | SQL の読みやすさと配置。 |
| フォントサイズ | `12`～`24` | `12` | エディターのテキスト サイズ。 |
| 行の高さ | `1.1`～`2.2` | `1.5` | SQL 行間の垂直方向の間隔。 |
| 行番号 | ` On `、` Off ` | ` On ` | 行番号を表示するかどうか。 |
| ミニマップ | ` On `、` Off ` | ` Off ` | モナコのミニマップが表示されるかどうか。 |
| ワードラップ | ` On `、` Off ` | ` On ` | 長い SQL 行がエディタ内で折り返されるかどうか。 |
| 折りたたみ | ` On `、` Off ` | ` On ` | SQL ブロックを折りたたむことができるかどうか。 |
| クエリ制限 | `100`, `200`, `500`, `1000`, `2000`, `5000` | `200` | クエリのプレビューと分析ワークフローのデフォルトの行制限。 |

## アプリのテーマ設定

Dory にはアプリケーション レベルのカラー テーマも含まれています。

| グループ | テーマ |
| --- | --- |
| デフォルト | デフォルト、青、緑、琥珀、液体 |
| スケーリング済み | デフォルトのスケーリング、青のスケーリング、液体のスケーリング |
| 等幅 | モノスケール |

共有画面、デモ、チームのウォークスルーには、抑制されたテーマを選択してください。密な SQL とテーブル スキャンが主なワークフローである場合は、等幅バリアントを選択します。

## クエリ制限のガイダンス

| 使用事例 | 推奨される制限 |
| --- | --- |
| スキーマの探索 | `100`または`200` |
| ダッシュボードのプロトタイピング | `200`または`500` |
| データ品質チェック | `500`または`1000` |
| 輸出類似検査 | 必要な場合のみ`2000`または`5000` |

制限を低くすると、Dory がより速く感じるようになり、偶発的な大規模スキャンが減少します。分析集計の場合は、結果の制限を上げるよりも SQL レベルのグループ化を優先します。

## 接続設定

接続設定はデータベース ドライバーによって異なりますが、ほとんどのチームは次の決定を標準化する必要があります。

| エリア | おすすめ |
| --- | --- |
| 名前 | ` prod-analytics-readonly `など、環境とデータベースの目的を含む名前を使用します。 |
| 資格 | 分析接続には読み取り専用データベース ユーザーを使用します。 |
| ホストアクセス | Docker デプロイメントがコンテナー内からデータベース ホストに到達できることを確認します。 |
| ローカルファイル | SQLite と DuckDB の場合は、Docker で実行するときに絶対パスを使用し、含まれているディレクトリをマウントします。 |
| SSL | 便宜上検証を無効にするのではなく、データベース プロバイダーのSSL要件に従ってください。 |

## AI 設定

AI の動作は、展開時に選択されたプロバイダーとモデルによって制御されます。詳細なリファレンスについては、[AI プロバイダー](/ja/docs/reference/ai-providers) を参照してください。

チームで使用する場合は、次の文書を作成します。

- 承認されたプロバイダーとモデル。
- どの環境で AI 機能を使用できるか。
- プロンプトで実稼働スキーマ名を許可するかどうか。
- AI が生成した SQL を実行する前に予想されるレビュー プロセス。

## 保存されたクエリ設定

保存されたクエリは、チームが名前とフォルダーの規則に同意する場合に最も役立ちます。

| 大会 | 例 |
| --- | --- |
| ドメインによるプレフィックス | ` billing - monthly revenue ` |
| 穀物を含む | ` orders - daily count ` |
| 曖昧な場合はデータベースを含める | ` warehouse.users - active users ` |
| 下書きを分けて保存する | ` drafts / migration checks ` |

## 推奨されるチームのデフォルト

- 新規ユーザーのクエリ制限を`200`に保ちます。
- オンボーディングとデモではワードラップをオンにしておきます。
- 共有接続には読み取り専用の資格情報を使用します。
- 1 回限りのスクリーンショットを共有する代わりに、再利用可能な SQL を保存します。
- 実行前に AI によって生成された SQL(特に結合とフィルター) を確認します。
