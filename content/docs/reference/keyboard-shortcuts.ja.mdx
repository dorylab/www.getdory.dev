# キーボード ショートカット

Dory のキーボード ショートカットは、高速な SQL 探索のために最適化されています。クエリを作成し、AI に助けを求め、実行し、結果を検査し、キーボードから離れることなく有用なバージョンを保存します。

macOS では、このページでは`Cmd/Ctrl `と記載されている場所で、` Cmd `を使用します。 Windows および Linux では、` Ctrl `を使用します。

## SQL エディター

| ショートカット | アクション | 注意事項 |
| --- | --- | --- |
| ` Cmd/Ctrl + Enter ` | クエリの実行 | アクティブな SQL タブを実行します。テキストが選択されている場合は、エディター フローでサポートされている場合、その選択内容をクエリ コンテキストとして使用します。 |
| ` Cmd/Ctrl + Shift + F ` | 形式 SQL | アクティブなエディターで SQL をフォーマットします。 |
| ` Cmd/Ctrl + S ` | SQL を保存 | 現在の SQL テキストの保存ダイアログを開きます。 |
| ` Cmd/Ctrl + Alt + L ` | 新しい SQL タブ | 新しいクエリタブを作成してアクティブにします。 |
| 空のエディター行に`/` | インラインアスク AI | 現在の行でインライン AI プロンプトを開きます。 |
| 他の場所で`/` | スラッシュを入力してください | 現在の行にすでに SQL テキストが含まれている場合は、通常どおりスラッシュが挿入されます。 |

## AI パネル

| ショートカット | アクション | 注意事項 |
| --- | --- | --- |
| ` Cmd/Ctrl + I ` | AI チャットパネルの切り替え | アクティブな SQL タブの SQL コンソール AI パネルを開閉します。 |
| `/`編集可能なコントロールの外側 | インラインで AI に聞く | アクティブなタブが SQL タブで、フォーカスがまだ別の入力内にない場合に機能します。 |

## 結果テーブル

| ショートカット | アクション | 注意事項 |
| --- | --- | --- |
| ` Cmd/Ctrl + F ` | 検索結果 | 結果テーブルがアクティブな場合、結果テーブル検索を開きます。 |
| ` Cmd/Ctrl + C ` | 選択したセルをコピーする | 選択した結果セルを仮想化テーブルからコピーします。 |

## 保存されたクエリ

| ショートカット | アクション | 注意事項 |
| --- | --- | --- |
| ` Cmd/Ctrl + S ` | 現在の SQL を保存する | これは、クエリに名前を付け、整理し、共有するのに十分な再利用可能になった場合に使用します。 |

## 実践的なワークフロー

1. ` Cmd/Ctrl + Alt + L `を押してクリーンタブを開始します。
2. 空行に`/`と入力して、Dory AI にクエリ ドラフトを要求します。
3. SQL を確認し、` Cmd/Ctrl + Shift + F `を押してフォーマットします。
4. ` Cmd/Ctrl + Enter `を押してクエリを実行します。
5. ` Cmd/Ctrl + F `を使用して結果テーブルを検索します。
6. ` Cmd/Ctrl + S `を押して、完成したクエリを保存します。

## トラブルシューティングのショートカット

| 問題 | 修理 |
| --- | --- |
| 結果検索の代わりにブラウザ検索が開く | 結果テーブル内をクリックし、もう一度` Cmd/Ctrl + F `を押します。 |
| `/`は AI を開く代わりにスラッシュを入力します | 空のエディター行に移動するか、フォーカスが別の入力内にないことを確認してください。 |
| ` Cmd/Ctrl + S `がブラウザーの保存をトリガーします | SQL コンソール ページがフォーカスされていることを確認します。 |
| ショートカットがOSの設定と競合する | Dory がキーを受け取る前に OS またはブラウザのショートカットがキーを傍受した場合は、OS またはブラウザのショートカットを変更します。 |
