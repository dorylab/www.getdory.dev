# v0.7.1

[GitHub でソースを表示](https://github.com/dorylab/dory/blob/main/release-notes/v0.7.1.md)

## 何が変わったのか
＃＃＃ 特徴
- **電子メール検証の切り替え**: サインアップ中の電子メール検証を有効または無効にするための`NEXT_PUBLIC_REQUIRE_EM AI L_VERIFICATION `および` EM AI L_FROM `環境変数を追加しました。対応する UI およびバックエンドのサポートも含まれます。
- **アップグレード前のワークスペース バックアップ通知**: 英語、スペイン語、日本語、中国語にローカライズされた v0.8.0 にアップグレードする前にワークスペースをバックアップするようユーザーに促す目立つアプリ内アラートが導入されました。
- **匿名ユーザーの回復**: ゲスト ユーザーはセッションを回復し、サインイン後にログイン アカウントにシームレスに移行できるようになりました。これにより、保存されていない SQL クエリ、保存されたクエリ、フォルダー、および Copilot 履歴が保持されます。
- **ゲスト用の完全な SQL コパイロット**: 匿名ユーザーは、SQL コパイロット (SQL 生成、テーブル モード、保存機能を含む) に完全にアクセスできるようになり、オンボーディングと探索が向上しました。
### 修正
- **PGlite ブラウザの互換性**: ブラウザ環境で PGlite を初期化する際のランタイム エラーを修正しました。
- **サイドバー レイアウトの回帰**: 視覚的な配置に影響を与えるアプリケーション サイドバーの不正なマージンを修正しました。
- **デスクトップの安定性**: 起動時のバージョンの不安定性の問題を解決するために、デスクトップを v41 から v39.x にロールバックしました。
- **デスクトップ ブートストラップのハング**: ブートストラップの完了を適切に処理し、同時起動を防止することで、デスクトップ アプリの起動中のハングを解決しました。
### 改善点
- **データ移行を伴う PGlite のアップグレード**: 自動化された安全なデータ移行パスにより、PGlite が 0.2.17 (PostgreSQL 16) から 0.4.3 (PostgreSQL 17) にアップグレードされ、既存のワークスペース データがすべて保持されます。
- **スタンドアロン デスクトップ ビルド**:` better-sqlite3`の具体化、` node_modules `シンボリックリンクの解決、および必要な Web ` node_modules `のコピーにより、スタンドアロン アプリのパッケージ化の信頼性が向上しました。
