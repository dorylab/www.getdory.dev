# v0.3.7

[GitHub でソースを表示](https://github.com/dorylab/dory/blob/main/release-notes/v0.3.7.md)

## 何が変わったのか
### 修正
- 読み取り専用の SQL ステートメントを生成する際の無限の再試行を防ぐために、SQL コパイロットの再試行ロジックを修正しました。
- ツールチップの配置やコード ブロックのレンダリングなど、Copilot で生成された SQL 結果コンポーネントのスタイルの不一致を解決しました。
- スキーマ コンテキスト プロンプト内の列メタデータの欠落によって引き起こされる AI アクションの失敗を修正しました。
- データベース概要ページのレイアウトの問題を修正し、メタデータ カードとツールヒントの視覚的な配置を改善しました。
- エンドツーエンド テストおよび接続フロー UI での接続確立エラーを修正しました。
- アプリのサイドバー アイコンの位置ずれと、ナビゲーション セクション間での更新ボタンのスタイルの不一致に対処しました。
### テストと CI の改善
- SQL コンソール、接続、認証フローをカバーする包括的なエンドツーエンド テストを追加しました。
- 自動化されたE2Eテスト実行のための新しい GitHub Actions ワークフローを導入しました。
- `.tmp `ディレクトリのフィルタリングや認証状態の処理など、複数の Playwright 設定の問題を修正しました。
- テスト ヘルパーと認証セットアップの対象を絞った修正により、不安定なE2Eテスト スイート (ログイン、ワークベンチ、接続セットアップなど) を安定化しました。
### ドキュメントとツール
- 廃止された環境変数 (` BETTER_AUTH_URL `、` DB_TYPE `) を削除するために、` README.md `の Docker run コマンドを更新しました。
- 専用の GitHub Action と CLI スクリプトを備えた AI を活用したリリース ノート ジェネレーターを追加しました。
