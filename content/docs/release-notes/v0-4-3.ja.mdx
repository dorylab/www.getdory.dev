# v0.4.3

[GitHub でソースを表示](https://github.com/dorylab/dory/blob/main/release-notes/v0.4.3.md)

## 何が変わったのか
＃＃＃ 特徴
- **組織プラグインの統合**: コア認証とチーム処理ロジックを専用の組織プラグインに移行し、Web アプリ全体でよりスケーラブルでモジュール式の複数組織のサポートを可能にします。
### 改善点
- **チーム コンテキストの処理の簡素化**: 認証フロー (デスクトップ認証の消費/ファイナライズ)、APIルート、およびクライアント側の認証ユーティリティ全体にわたる`defaultTeamId `への依存を削除しました。これにより、現在の組織またはチームを解決する際の曖昧さが軽減され、一貫性が向上しました。
### 修正
- **データベース移行の信頼性**: 特に組織関連のスキーマ変更後、PostgreSQL および pglite 環境全体で一貫したスキーマ状態を確保するために、Drizzle 移行ツールが更新され、修復スクリプトが追加されました。
### ドキュメント
- **README更新**: セットアップ手順、アーキテクチャに関するメモ、組織を意識した新しいワークフローの使用方法のガイダンスなど、ドキュメントの明瞭さに対する複数の反復的な改善。
