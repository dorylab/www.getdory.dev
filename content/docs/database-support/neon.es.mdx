# Neón

<img src="/icons/databases/neon.svg" alt="Neon logo" width="48" height="48" />

Neon es una plataforma PostgreSQL sin servidor. Dory se conecta a Neon a través de capacidades compatibles con PostgreSQL, lo que lo hace útil para el desarrollo de la nube, bases de datos de sucursales, análisis de productos y flujos de trabajo de equipo.

## Casos de uso más adecuados

- Consulta de datos comerciales en proyectos Neon.
- Analizar ramas de desarrollo, puesta en escena o producción.
- Generar SQL compatible con PostgreSQL con IA.
- Validar rápidamente los problemas de datos de la aplicación.
- Guarde consultas de Neon reutilizables para el equipo.

## Requisitos de conexión

Prepare la cadena de conexión de Neon o los detalles de la conexión, el proyecto, la rama, la base de datos, el nombre de usuario, la contraseña, la configuraciónSSLy los permisos del esquema. El neón comúnmente requiereSSL.

## Capacidades admitidas

| Capacidad | Soporte de neón |
| --- | --- |
| Consola SQL | Ejecución de SQL compatible con PostgreSQL. |
| Explorador | Esquemas, tablas, vistas, columnas y DDL. |
| Chat de IA | Generación, explicación y corrección de SQL basada en esquemas. |
| Gráficos y resultados | Análisis visual de resultados de consultas. |
| Consultas guardadas | Guardar SQL reutilizable. |

## Relación con PostgreSQL

Neon es PostgreSQL sin servidor. En Dory, Neon utiliza un comportamiento compatible con PostgreSQL, por lo que también se aplican la mayoría de las instrucciones de PostgreSQL, incluida la selección de esquemas, roles de solo lectura y revisión de IA SQL.

## FAQ

### ¿Por qué falla mi conexión de Neon?

Verifique la cadena de conexión, el host, la base de datos, el nombre de usuario, la contraseña, la configuraciónSSLy si el proyecto Neon está en pausa.

### ¿Cómo debo distinguir las bases de datos de las sucursales de Neon?

Utilice nombres de conexión que incluyan el proyecto y la rama, como `neon-prod-main ` o `neon-staging-preview `.
