# SQLite

<img src="/icons/databases/sqlite.svg" alt="SQLite logo" width="48" height="48" />

Dory は、開発、プロトタイプ、デモ、テスト データ、軽量分析用のローカル SQLite データベース ファイルをサポートしています。`.sqlite `または`.db `ファイルを開いたり、テーブルを参照したり、SQL を実行したり、結果を検査したり、クエリを保存したりできます。

## 最適な使用例

- ローカル開発データベースの検査。
- 製品プロトタイプとデモ データセット。
- 小規模なデータセットの探索。
- 再現可能なテスト データ クエリ。
- リモートデータベースを使用しない軽量分析。

## 接続要件

SQLite ファイルへの絶対パスを指定します。 Dory プロセスはファイルを読み取ることができる必要があります。 Docker で、ファイル ディレクトリをコンテナにマウントし、コンテナのパスを使用します。

## サポートされる機能

| 能力 | SQLiteのサポート |
| --- | --- |
| 接続 | 絶対パス`.sqlite `/`.db `ファイル接続。 |
| エクスプローラ | テーブル、列、および基本構造。 |
| SQL コンソール | SQLite SQL の実行と結果。 |
| AI チャット | スキーマに基づいた SQL の生成と説明。 |
| チャートと結果 | クエリ結果の視覚的な分析。 |
| 保存されたクエリ | 再利用可能な SQLite SQL を保存します。 |

## Docker で SQLite を使用する

Dory を Docker で実行する場合、SQLite ファイルをコンテナーにマウントする必要があります。 Dory 接続では、ホストのみのパスではなく、コンテナー パスを使用します。

## FAQ

### Dory が私の SQLite ファイルを見つけられないのはなぜですか?

パスが絶対パスであり、Dory が読み取り可能であることを確認します。 Docker で、ディレクトリがコンテナにマウントされていることを確認します。

### SQLite はチーム共有に適していますか?

SQLite は、ローカル ファイル、デモ、軽量分析に最適です。共有チーム環境の場合は、PostgreSQL、MySQL、ClickHouse、または別のサーバー データベースを検討してください。
