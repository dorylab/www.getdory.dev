# クリックハウス

<img src="/icons/databases/clickhouse.svg" alt="ClickHouse logo" width="48" height="48" />

ClickHouse は、Dory の最も深いデータベース統合の 1 つです。SQL コンソール、エクスプローラー、AI チャット、チャートに加えて、Dory は ClickHouse 固有の監視と権限管理を提供します。

## 最適な使用例

- 製品イベント分析。
- ログ、トレース、可観測性データの探索。
- 大規模な詳細テーブル分析。
- 遅いクエリとエラー クエリの調査。
- ClickHouse ユーザー、役割、および付与の管理。

## サポートされる機能

| 能力 | クリックハウスのサポート |
| --- | --- |
| SQL コンソール | ClickHouse SQL を作成、実行、反復します。 |
| エクスプローラ | データベース、テーブル、ビュー、列、DDL、パーティション キー、および並べ替えキーを参照します。 |
| AI チャット | ClickHouse スキーマ コンテキストを使用して SQL を生成、説明、修正、最適化します。 |
| チャートと結果 | クエリ結果を傾向、比較、詳細ビューに変換します。 |
| 監視 | 合計クエリ、遅いクエリ、エラー、P50/P95遅延、およびQPMを検査します。 |
| 特権 | ユーザー、ロール、グローバル権限、および範囲指定された付与を管理します。 |

## 接続要件

ClickHouse のホスト、ポート、データベース、ユーザー名、パスワード、TLS設定、および必要なSSHトンネルの詳細を準備します。 ClickHouse クラウド接続には通常、SSLが必要です。

## 推奨フロー

1. [接続](/ja/docs/core-features/connections)を開きます。
2. 「クリックハウス」を選択します。
3. ホスト、ポート、HTTPポート、データベース、ユーザー名、およびパスワードを入力します。
4. デプロイメントにTLSが必要な場合は、SSLを有効にします。
5. 接続をテストして保存します。
6. SQL コンソールまたはエクスプローラーを開きます。

## AI クエリのヒント

AI に ClickHouse SQL の生成を依頼する場合は、時間範囲、メトリクス、ターゲット テーブル、グループ化ディメンション、および`toStartOfHour `、` quantile `、` uniq `などの関数設定を含めます。

## クリックハウス監視

Dory は、接続されているユーザーが必要なシステム テーブルを読み取ることができるときに、ClickHouse モニタリング ビューを表示できます。これらのビューを使用して、最近のクエリ、遅いクエリ、エラー、待機時間、およびQPMを検査します。

## ClickHouse 権限管理

権限が利用可能な場合、Dory は、ClickHouse 環境のユーザー、ロール、グローバル権限、および範囲指定された付与のレビューを支援できます。

## FAQ

### ClickHouse 接続が失敗するのはなぜですか?

ホスト、ポート、ユーザー名、パスワード、TLS設定、およびネットワーク許可リストを確認してください。 ClickHouse Cloud の場合、通常、SSLが必要です。

### 監視が空なのはなぜですか?

現在のユーザーに ClickHouse システム テーブルを読み取る権限がないか、選択された時間範囲にクエリ レコードが含まれていない可能性があります。

### AI が生成した ClickHouse SQL はなぜ編集が必要なのでしょうか?

ClickHouse の関数と構文は、バージョンと展開によって異なる場合があります。生成された SQL をスキーマと照合して検証し、運用分析に使用する前に小さなプレビューを実行します。
